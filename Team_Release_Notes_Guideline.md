# 글로벌 표준 팀 릴리스 노트 작성 가이드라인 (Team Release Notes Guideline)

글로벌 선도 소프트웨어 기업(GitHub, Slack, Atlassian, Raycast 등)의 릴리스 노트 모범 사례를 분석하여 우리 팀에 적합한 릴리스 노트 작성 기준과 템플릿을 수립했습니다.

---

## 1. 릴리스 노트의 목적 및 핵심 원칙

글로벌 기업들은 릴리스 노트를 단순한 '기술 업데이트 요약'이 아니라 **사용자와의 커뮤니케이션 및 제품의 가치를 홍보하는 마케팅 도구**로 활용합니다.

### 🎯 3대 핵심 원칙
1. **사용자 중심 (User-Centric)**: 개발자의 언어(Jira 티켓 번호, 내부 아키텍처 변경)가 아닌 **사용자의 언어**로 작성합니다. *"우리가 무엇을 개발했는가"* 보다 *"이것이 사용자에게 어떤 가치와 혜택을 주는가"*에 집중합니다.
2. **명확성과 스캐너블 (Clear & Scannable)**: 텍스트 벽(Wall of text)을 피하고, 헤딩, 불릿 포인트, 짧은 문단을 사용하여 대각선으로 훑어봐도 핵심을 파악할 수 있게 작성합니다.
3. **시각적 요소 활용 (Visual Aids)**: 백문이 불여일견. 새로운 기능이나 변경된 UI는 스크린샷, GIF, 짧은 비디오 클립을 적극 활용하여 설명합니다.

---

## 2. 권장하는 작성 구조 (구조화 및 분류)

복잡한 업데이트 내용을 논리적으로 그룹화하여 사용자가 자신에게 필요한 정보를 빠르게 찾을 수 있도록 합니다.

*   🚀 **New Features (새로운 기능)**: 제품에 완전히 새롭게 추가된 기능. 가장 먼저 노출하여 기대감을 줍니다.
*   ✨ **Improvements / Enhancements (개선 사항)**: 기존 기능의 사용성 강화, 성능 최적화, UI 변경 등.
*   🐛 **Bug Fixes (버그 수정)**: 사용자에게 불편을 줬던 문제의 해결. (단, 사용자가 인지하지 못하는 마이너한 백엔드 버그 수정은 제외하는 것이 좋습니다.)
*   🗑️ **Deprecations (지원 중단)**: 곧 사라지거나 변경될 기능에 대한 사전 안내 및 대안 제시.

---

## 3. 기업별 모범 사례 벤치마킹 포인트

*   **Slack**: 기술적인 내용을 유머와 위트 있는 어조로 풀어내어 읽는 재미를 제공. 플랫폼별(Windows, Mac, Mobile)로 변경 사항을 명확히 구분.
*   **GitHub / Atlassian (Jira)**: 개발자 친화적이면서도 체계적인 마크다운 포맷 활용. 버전 히스토리(Changelog) 관리가 명확하며, 관련 문서나 상세 가이드로 이어지는 문서 링크(Learn more)를 적극 활용.
*   **Raycast / Loom**: 텍스트 설명보다 시각적 자료(GIF, 데모 영상)를 전면에 배치하여 새로운 기능의 "사용 감각"을 직관적으로 전달.

---

## 4. 우리 팀을 위한 릴리스 노트 템플릿

위의 원칙을 바탕으로 우리 팀이 사용할 수 있는 마크다운 기반의 표준 템플릿입니다.

```markdown
# 🚀 Release Notes: [제품명/모듈명] vX.Y.Z
**출시일**: YYYY년 MM월 DD일

> **한 줄 요약**: 이번 릴리스의 가장 큰 변화나 사용자 혜택을 1~2문장으로 요약합니다. (예: "더 빠르고 직관적인 워크플로우 빌더를 경험해 보세요.")

---

## 🚀 새로운 기능 (New Features)
새롭게 추가된 주요 기능을 설명합니다.

* **[기능명]**: 
  * **무엇이 바뀌었나요?**: 기능에 대한 간략한 설명.
  * **왜 중요한가요?**: 이 기능이 사용자에게 주는 혜택 (예: 작업 시간 단축, 보안 강화).
  * 💡 *Tip: 여기에 GIF나 스크린샷을 삽입하세요.*
  * [상세 가이드 보기](링크)

## ✨ 개선 사항 (Improvements)
기존 기능의 성능, UI/UX 등 변경된 사항을 나열합니다.

* **[개선 항목명]**: 이전에는 [불편했던 점]이었으나, 이제는 [개선된 방식]으로 작동하여 [기대 효과]를 얻을 수 있습니다.
* **[성능/UI 항목명]**: 화면 로딩 속도가 약 40% 향상되었습니다. / 메뉴 네비게이션이 더 직관적으로 재배치되었습니다.

## 🐛 버그 수정 (Bug Fixes)
사용자가 겪었던 불편함이 어떻게 해결되었는지 작성합니다. (티켓 번호 위주 나열 지양)

* **[버그 현상] 해결**: 특정 상황에서 [발생하던 문제]가 더 이상 발생하지 않도록 수정했습니다.
* **[버그 현상] 해결**: [기능] 사용 시 비정상 종료되던 문제를 해결하여 앱 안정성을 높였습니다.

## ⚠️ 알려진 문제 및 지원 중단 안내 (Known Issues & Deprecations)
* **알려진 문제**: 현재 인지하고 있는 버그 현상과 임시 우회(Workaround) 방법 안내. (예: "버전 3.2.1에서 수정될 예정입니다.")
* **지원 중단**: 더 이상 지원하지 않는 기능과 대안 기능 안내.
```

## 5. 단계별 체크리스트 (릴리스 전 최종 확인)

- [ ] 철저히 **사용자의 언어**로 작성되었는가? (내부 개발 용어 제거)
- [ ] 긴 문장이 없고 **스캐너블(Scannable)**하게 불릿 포인트가 잘 활용되었는가?
- [ ] 주요 변경 사항에 대해 **시각 자료(이미지/GIF)**가 첨부되었는가?
- [ ] 사용자가 더 깊이 알아볼 수 있는 **참고 문서/가이드 링크**가 포함되었는가?
